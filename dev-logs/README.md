# 开发日志规范

## 日志文件命名

每天一个文件，文件名格式：`YYYY-MM-DD.md`

示例：`2026-06-01.md`

## 更新时机

- **每日结束时**：必须更新当日日志
- **遇到阻塞问题时**：立即更新（不必等当天结束）
- **每个 Phase 完成时**：在日志中标记 Phase 完成

## 日志内容要求

每个日志文件必须包含以下五个部分：

1. **今日完成** — 列出今日完成的具体任务，附带耗时和产出的文件路径
2. **遇到的问题** — 记录遇到的问题、状态（已解决/待解决/阻塞）、解决方案
3. **明日计划** — 列出明日计划完成的任务
4. **当前风险** — 项目当前面临的风险及影响程度
5. **项目状态** — 当前 Phase、完成百分比、测试通过数量

## 状态标记说明

- `✅` 已完成
- `🏃` 进行中
- `❌` 已取消
- `🚫` 阻塞
- `⏳` 等待中
