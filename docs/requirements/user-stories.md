# 用户故事

## 角色 1：数据库工程师 (DBA) — 小王

### US-001: 注册数据库连接
> 作为 DBA，我希望能够添加并保存多个数据库连接配置，以便后续快速切换源库和目标库。

**验收条件**：
- 输入连接信息（名称、类型、主机、端口、用户名、密码、库名）
- 点击"测试连接"验证可用性
- 连接配置加密保存到本地

### US-002: 扫描单条 SQL
> 作为 DBA，我希望输入一条 SQL，系统能告诉我这条 SQL 在目标数据库中是否存在兼容性问题。

**验收条件**：
- 选择源数据库类型和目标数据库类型
- 输入 SQL 并点击扫描
- 显示每条兼容性问题的详细信息（规则、严重程度、建议）

### US-003: 批量扫描 SQL 文件
> 作为 DBA，我有一个包含数百条建表语句的 .sql 文件，我希望一次性扫描全部 SQL 的兼容性。

**验收条件**：
- 选择或拖拽 .sql 文件
- 显示全部扫描结果，按严重程度排序
- 支持导出扫描结果为报告

### US-004: 查看自动转换建议
> 作为 DBA，对于可以自动转换的 SQL，我希望看到转换后的结果，不需要手动逐条修改。

**验收条件**：
- 对 AUTO_CONVERTIBLE 级别的 SQL，显示"原文 ↔ 转换后"对比
- 支持复制转换后的 SQL
- 支持批量导出全部转换结果

### US-005: 导出迁移报告
> 作为 DBA，我需要将扫描结果以正式报告形式导出，用于向领导汇报迁移风险评估。

**验收条件**：
- 支持 HTML / PDF / Word 三种格式
- 报告包含：概览统计、按严重程度分组的问题列表、转换建议
- 报告包含图表（按规则分类的问题分布、按文件的问题分布）

## 角色 2：后端开发者 — 小李

### US-006: 扫描项目源码
> 作为 Java 开发者，我的项目中有很多 MyBatis Mapper XML 和 Java 代码中拼接的 SQL，我希望能一次性扫描整个项目找出所有需要修改的 SQL。

**验收条件**：
- 选择项目根目录
- 自动扫描所有 .java 和 .xml 文件
- 识别其中的 SQL 语句并给出兼容性分析
- 每个问题指向具体的文件路径和行号

### US-007: 查看 SQL 修改建议
> 作为开发者，我希望看到每条不兼容 SQL 的具体修改建议和示例，方便我在代码中修改。

**验收条件**：
- 显示原始 SQL 和修改后的 SQL
- 显示规则描述和转换示例
- 显示文件路径和行号，方便在 IDE 中定位

## 角色 3：迁移顾问/架构师 — 张工

### US-008: 创建迁移任务
> 作为架构师，我希望创建一个包含源库和目标库的迁移任务，系统能全面分析并给出迁移评估。

**验收条件**：
- 创建任务，指定源连接和目标连接
- 任务执行后显示：总问题数、按严重程度分布、风险评分
- 支持保存任务并在后续重新查看/更新

### US-009: 查看风险评估
> 作为架构师，我需要一个整体的迁移风险评估，包括问题数量、严重程度分布、预估工作量等。

**验收条件**：
- 显示风险评分（0-100 分）
- 显示按分类统计的问题数量（函数/语法/数据类型/DDL）
- 显示可自动转换 vs 需人工处理的比例
