# 产品简介与定位

## 产品名称

**Database Migration Assistant (DMA)** — 数据库迁移助手

## 产品定位

面向数据库工程师、DBA 和后端开发者的数据库迁移与兼容性分析工具。解决企业在数据库迁移（尤其是国产化替代）过程中 SQL 不兼容、存储过程转换、项目源码扫描等痛点。

## 目标用户

| 角色 | 典型场景 | 核心需求 |
|------|---------|---------|
| **数据库工程师 / DBA** | 负责将现有数据库迁移到新平台（如 PG、达梦） | 批量 SQL 兼容性扫描、存储过程/函数分析、迁移风险评估 |
| **后端开发者** | 修改应用代码以适配新数据库 | Java/XML Mapper 中的 SQL 检测、自动修复建议 |
| **迁移顾问 / 架构师** | 评估迁移可行性、制定迁移计划 | 全量扫描报告、风险评估、AI 辅助决策 |

## 核心价值

1. **自动化** — 自动识别不兼容 SQL，自动生成转换建议
2. **全面性** — 覆盖数据库对象 + 应用源码两个维度
3. **可扩展** — 规则引擎支持自定义规则和社区共享
4. **企业级** — 报告导出、风险管理、AI 辅助决策

## 产品形态

| 形态 | 定位 | 优先级 |
|------|------|--------|
| **Windows 桌面端 (JavaFX)** | 独立应用，完整功能 | 🥇 MVP 优先 |
| **IntelliJ IDEA 插件** | IDE 内集成体验，实时检测 | 🥈 后续迭代 |
| **Web SaaS（远期）** | 云端服务，团队协作 | 🥉 商业化阶段 |

## 商业化策略

- **IDEA 插件**：免费（获客渠道）
- **桌面端基础功能**：免费（SQL 扫描、转换、报告）
- **桌面端高级功能**：付费（AI 迁移顾问、批量迁移、团队协作、规则市场）
- **企业版**：私有化部署 + 定制规则 + 技术支持
