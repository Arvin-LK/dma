# 功能规格说明

## 1. 数据库连接管理

### 1.1 创建连接

| 属性 | 说明 |
|------|------|
| **输入** | 连接名称、数据库类型（枚举）、主机地址、端口、用户名、密码、数据库名 |
| **处理** | 1. 校验必填字段 <br> 2. 使用 AES 加密密码 <br> 3. 保存到 SQLite |
| **输出** | 连接 ID |
| **异常** | 必填字段为空 → `ValidationException` <br> 连接名重复 → `DuplicateConnectionException` |

### 1.2 测试连接

| 属性 | 说明 |
|------|------|
| **输入** | 连接 ID 或完整连接参数 |
| **处理** | 1. 使用 Druid 创建临时连接池 <br> 2. 执行 `SELECT 1` <br> 3. 关闭连接池 |
| **输出** | 成功/失败 + 延迟毫秒数 |
| **异常** | 连接超时 → `ConnectionTimeoutException` <br> 认证失败 → `AuthenticationException` |

### 1.3 连接列表

| 属性 | 说明 |
|------|------|
| **输入** | 分页参数（页码、每页条数） |
| **处理** | 从 SQLite 查询，按创建时间倒序 |
| **输出** | 连接列表（不含密码） + 总数 |

## 2. SQL 兼容性扫描

### 2.1 单条 SQL 扫描

| 属性 | 说明 |
|------|------|
| **输入** | SQL 文本、源数据库类型、目标数据库类型 |
| **处理** | 1. 使用 JSQLParser 解析 SQL 为 AST <br> 2. 加载对应的兼容性规则 <br> 3. 逐规则匹配 AST <br> 4. 对每条匹配返回兼容性结果 |
| **输出** | `List<CompatibilityResult>` 包含：规则代码、兼容性级别、匹配位置、原文片段、建议替换 |
| **边界条件** | SQL 解析失败 → 返回 PARSE_ERROR 结果（不阻断）<br> 空 SQL → 返回空列表 <br> SQL 超过 10000 字符 → 截断分析并警告 |

### 2.2 SQL 文件扫描

| 属性 | 说明 |
|------|------|
| **输入** | 文件路径、源数据库类型、目标数据库类型 |
| **处理** | 1. 读取文件 <br> 2. 按 `;` 分割为多条 SQL <br> 3. 逐条调用单条扫描 |
| **输出** | 合并的结果列表，每条结果附加文件路径 |
| **边界条件** | 文件不存在 → `FileNotFoundException` <br> 文件超过 10MB → 拒绝处理 |

### 2.3 兼容性级别定义

| 级别 | 含义 | 示例 |
|------|------|------|
| `COMPATIBLE` | 完全兼容，无需修改 | `SELECT * FROM users` 在 MySQL 和 PG 均可 |
| `AUTO_CONVERTIBLE` | 可自动转换 | `IFNULL(a,b)` → `COALESCE(a,b)` |
| `MANUAL_REVIEW` | 需要人工审核 | 存储过程语法差异 |
| `INCOMPATIBLE` | 不兼容且无法自动转换 | 使用了目标库不支持的特性 |
| `PARSE_ERROR` | SQL 无法解析 | 语法错误的 SQL |

## 3. SQL 自动转换

### 3.1 转换方式

| 属性 | 说明 |
|------|------|
| **输入** | SQL 文本、源数据库类型、目标数据库类型 |
| **处理** | 1. 解析 SQL 为 AST <br> 2. 加载规则并匹配 <br> 3. 对 AUTO_CONVERTIBLE 级别结果，应用转换策略 <br> 4. 返回转换后的 SQL |
| **输出** | `ConvertResult` 包含：原始 SQL、转换后 SQL、变更列表 |
| **边界条件** | 无匹配规则 → 返回原始 SQL <br> 多个规则匹配同一位置 → 优先应用严重程度高的 |

### 3.2 支持的转换类型（MVP）

| 转换类型 | 示例 |
|---------|------|
| 函数名称替换 | `IFNULL(a,b)` → `COALESCE(a,b)` |
| 分页语法 | `LIMIT 10,20` → `LIMIT 20 OFFSET 10` |
| 数据类型 | DDL 中 `DATETIME` → `TIMESTAMP` |
| 标识符引用 | `` `col` `` → `"col"` |
| 自增列 | `AUTO_INCREMENT` → `SERIAL` |
| NULL 处理 | `NVL(a,b)` → `COALESCE(a,b)` |

## 4. 源码扫描

### 4.1 Java 源码扫描

| 属性 | 说明 |
|------|------|
| **输入** | 项目根目录路径 |
| **处理** | 1. 递归遍历所有 .java 文件 <br> 2. 正则匹配字符串字面量中的 SQL 关键词 <br> 3. 提取疑似 SQL 的字符串 <br> 4. 对提取的 SQL 执行兼容性分析 |
| **输出** | 扫描结果列表，每项包含文件路径、行号、原始 SQL、分析结果 |
| **边界条件** | 动态拼接的 SQL（含变量）→ 标记为 MANUAL_REVIEW <br> 非 SQL 字符串被误识别 → 允许人工忽略 |

### 4.2 MyBatis Mapper XML 扫描

| 属性 | 说明 |
|------|------|
| **输入** | 项目根目录路径 |
| **处理** | 1. 递归遍历所有 *Mapper.xml 文件 <br> 2. XML 解析，提取 `<select>/<insert>/<update>/<delete>` 标签内容 <br> 3. 处理 `${}` 和 `#{}` 占位符 <br> 4. 对提取的 SQL 执行兼容性分析 |
| **输出** | 扫描结果列表，每项包含 XML 文件路径、标签 ID、原始 SQL、分析结果 |
| **边界条件** | 含 `<if>/<foreach>` 动态标签 → 标记为 MANUAL_REVIEW <br> 纯 `${}` 拼接 → 标记为高风险 |

## 5. 报告生成

### 5.1 报告内容结构

1. **封面** — 任务名称、数据库类型、生成时间
2. **概览摘要** — 总问题数、按严重程度分布、风险评分、可自动转换比例
3. **统计图表** — 按规则分类的饼图、按文件分布的热力图
4. **详细问题列表** — 每项：文件位置、SQL原文、转换建议、严重程度、处理建议
5. **建议汇总** — 按优先级排列的迁移建议清单

### 5.2 导出格式

| 格式 | 技术实现 | 特点 |
|------|---------|------|
| HTML | Thymeleaf 模板渲染 | 可在浏览器查看、支持交互式筛选 |
| PDF | Flying Saucer 将 HTML 转 PDF | 正式报告、不可编辑 |
| Word | Apache POI 生成 .docx | 可编辑、适合团队协作 |

## 6. 迁移任务管理

### 6.1 任务生命周期

```
PENDING → RUNNING → COMPLETED
                 → FAILED
```

### 6.2 任务属性

| 字段 | 说明 |
|------|------|
| 任务名称 | 用户自定义 |
| 源连接 | 关联的数据库连接 ID |
| 目标连接 | 关联的数据库连接 ID |
| 状态 | PENDING / RUNNING / COMPLETED / FAILED |
| 统计 | totalIssues, resolvedIssues, riskScore |
| 时间 | createdAt, completedAt |
